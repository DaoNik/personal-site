# Чеклисты

- Новая фича: уточнить требования, создать библиотеку/папку в `client-features` при необходимости, описать публичный API. Проверить маршруты/навигацию.
- Разработка: использовать Nx команды (`nx serve client`, `nx serve api`), избегать прямых cli. Соблюдать архитектурные и код-стандарты.
- Контракты: согласовать REST endpoints, обновить общие интерфейсы/типы и переиспользовать на фронте/бэке.
- Проверки перед пушем: `nx lint <project>`, `nx test <project>` (если есть), сборка целевого проекта. Убедиться, что алиасы и импорты не ломают границы модулей.
- Документация: если добавлены правила/паттерны — кратко обновить `docs/` при необходимости.
